# Spark to Code

语言： [English](README.md) | **中文** | [Français](README.fr.md) | [Deutsch](README.de.md)

这不是源码仓库。  
这是一个连接创意、痛点与开发者的协作社区。

目标很简单：让好想法被看见、被验证、被实现。

## 仓库目的

- 对创意发起者：把真实痛点说出来，不让灵感沉没在私人笔记里。
- 对开发者：找到有价值的 Side Project 与协作者。
- 对社区：更早发现已有工具，减少重复造轮子。

## 这个仓库做什么

1. 需求集散地  
任何人都可以提交软件需求和具体痛点。

2. 组队协作区  
发起者可以招募协作者，并说明角色与技术栈需求。

3. 开发进度同步  
项目进入开发后，团队可以持续更新里程碑、阻塞项和进展。

4. 现成工具指路  
如果已有成熟方案，社区成员可以直接推荐链接。

## 参与机制：Issue-first, PR-second

### Issue-first

Issue 是主入口：
- `idea`：提交新想法或痛点
- `looking-for-solution`：请求现有软件推荐
- `building-update`：同步项目/团队开发进展
- 评论：补充范围、推荐工具、表达组队意愿

### PR-second

PR 是可选入口，仅用于维护仓库内容：
- 更新文档与模板
- 维护 `PROJECTS.md` 等精选清单

不要把 PR 作为提交想法的主要方式。

## 谁可以发布

所有人都可以。  
本仓库不仅面向开发者，也欢迎非技术用户提需求。

## 社区规则

1. 创建新 Issue 时请使用模板。
2. 避免空泛描述，必须写清具体场景和约束。
3. 如果已有成熟替代方案，请附官方链接并建议关闭。
4. 想加入团队时，请说明角色、技术栈和可投入时间。
5. 发布在此的创意视为公开共享，请勿提交保密商业信息。

## 标签体系（英文）

| Label | 含义 |
| --- | --- |
| `idea` | 新想法或痛点 |
| `looking-for-solution` | 寻找现成工具推荐 |
| `team-wanted` | 招募协作者 |
| `building` | 正在开发 |
| `solved` | 已有方案或已实现 |
| `duplicate` | 与现有 Issue 重复 |
| `needs-info` | 缺少关键上下文 |
| `stale` | 长期无活动 |

## 维护者轻量流程

1. 每周执行一次 Triage：去重、打标、关闭已解决议题。
2. 鼓励开发中的团队在同一 Issue 持续更新进度。
3. 对信息不足议题标记 `needs-info`，长期无回应则关闭。

# Spark to Code

Langue : [English](README.md) | [中文](README.zh-CN.md) | **Français** | [Deutsch](README.de.md)

Ce dépôt n'est pas un dépôt de code source.  
C'est un espace de collaboration qui relie les idées, les problèmes réels et les développeurs.

Objectif : rendre les bonnes idées visibles, validées et mises en oeuvre.

## Objectif

- Pour les porteurs d'idées : partager des besoins réels avant qu'ils ne disparaissent dans des notes privées.
- Pour les développeurs : trouver des side projects utiles et des collaborateurs.
- Pour la communauté : découvrir les solutions existantes plus tôt et éviter de refaire la même chose.

## Rôle du dépôt

1. Hub de demandes  
Tout le monde peut publier un besoin logiciel concret.

2. Espace de collaboration  
Les auteurs peuvent recruter des profils en précisant rôle et stack.

3. Suivi de développement  
Les équipes peuvent publier jalons, blocages et avancement.

4. Découverte d'outils  
Si une solution mature existe déjà, la communauté peut la recommander directement.

## Modèle de participation : Issue-first, PR-second

### Issue-first

Les issues sont le canal principal :
- `idea` : nouvelle idée ou douleur utilisateur
- `looking-for-solution` : demande de recommandation d'outil existant
- `building-update` : mise à jour d'un projet en cours
- commentaires : affiner le périmètre, proposer des outils, rejoindre l'équipe

### PR-second

Les PR sont optionnelles et réservées à la maintenance du dépôt :
- mise à jour des documents et modèles
- maintenance des listes curatées comme `PROJECTS.md`

Ne pas utiliser les PR comme canal principal pour soumettre des idées.

## Qui peut publier

Tout le monde.  
Ce dépôt accepte les demandes des développeurs et des non-développeurs.

## Règles de communauté

1. Utiliser les modèles d'issues.
2. Éviter les demandes vagues : préciser le contexte et les contraintes.
3. Si une alternative mature existe, partager le lien officiel et proposer la fermeture.
4. Pour rejoindre une équipe, préciser le rôle, la stack et le temps disponible.
5. Les idées publiées ici sont considérées comme des concepts publics. Ne pas publier d'informations confidentielles.

## Système de labels (anglais)

| Label | Signification |
| --- | --- |
| `idea` | Nouvelle idée ou douleur utilisateur |
| `looking-for-solution` | Recherche de recommandations d'outils |
| `team-wanted` | Recrutement de collaborateurs |
| `building` | Projet en développement |
| `solved` | Résolu par un outil existant ou implémenté |
| `duplicate` | Doublon d'une issue existante |
| `needs-info` | Contexte insuffisant |
| `stale` | Aucune activité récente |

## Flux léger pour mainteneurs

1. Triage hebdomadaire : dédoublonner, labelliser, fermer les issues résolues.
2. Encourager les équipes actives à publier les avancées dans la même issue.
3. Marquer `needs-info` si nécessaire et fermer après longue inactivité.

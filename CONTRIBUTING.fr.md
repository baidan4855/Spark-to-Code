# Contributing to Spark to Code

Langue : [English](CONTRIBUTING.md) | [中文](CONTRIBUTING.zh-CN.md) | **Français** | [Deutsch](CONTRIBUTING.de.md)

Ce dépôt est un espace de collaboration, pas un dépôt de code source.  
L'objectif est de faire passer les idées de la douleur utilisateur à l'implémentation.

## Canaux de contribution

1. Ouvrir une issue (par défaut)
- Soumettre une nouvelle idée
- Demander des recommandations d'outils existants
- Partager l'avancement d'un projet en cours

2. Commenter les issues
- Ajouter des cas d'usage concrets
- Proposer des liens officiels vers des solutions matures
- Se proposer pour rejoindre une équipe

3. Ouvrir une PR (secondaire)
- Mettre à jour la documentation, les modèles et les listes
- Améliorer le workflow de gouvernance du dépôt

## Avant d'ouvrir une issue

1. Rechercher les issues existantes pour éviter les doublons.
2. Utiliser le bon formulaire d'issue.
3. Décrire clairement qui a le problème et dans quel contexte.

## Bonnes pratiques de collaboration

1. Rester précis et orienté action.
2. En cas de recommandation d'outil, inclure le lien officiel ou GitHub.
3. Pour rejoindre un projet, préciser :
- votre rôle
- votre stack technique
- votre disponibilité hebdomadaire

## Politique d'idées publiques

Les idées publiées ici sont considérées comme des concepts publics.  
Ne publiez pas d'informations confidentielles ou sensibles.

## Triage recommandé pour les mainteneurs

1. Triage hebdomadaire :
- appliquer les labels
- fusionner les doublons
- fermer les issues résolues avec références

2. Pour les projets actifs :
- demander des mises à jour de jalons et d'avancement dans la même issue

3. En cas d'informations manquantes :
- appliquer `needs-info`
- fermer après longue inactivité si les détails restent manquants

# Contributing to Spark to Code

Sprache: [English](CONTRIBUTING.md) | [中文](CONTRIBUTING.zh-CN.md) | [Français](CONTRIBUTING.fr.md) | **Deutsch**

Dieses Repository ist ein Kollaborationsraum und kein Quellcode-Repository.  
Ziel ist es, Ideen von realen Pain Points zur Umsetzung zu bringen.

## Beitragskanäle

1. Issue erstellen (Standard)
- Neue Idee einreichen
- Empfehlungen für bestehende Tools anfragen
- Fortschritt für laufende Projekte teilen

2. Issues kommentieren
- Konkrete Nutzungsszenarien ergänzen
- Offizielle Links zu reifen Lösungen teilen
- Interesse am Mitwirken zeigen

3. PR erstellen (sekundär)
- Dokumente, Templates und kuratierte Listen aktualisieren
- Governance- und Workflow-Prozesse verbessern

## Vor dem Erstellen eines Issues

1. Vorhandene Issues prüfen, um Duplikate zu vermeiden.
2. Das passende Issue-Formular verwenden.
3. Klar beschreiben, wer das Problem in welchem Kontext hat.

## Zusammenarbeit

1. Feedback konkret und umsetzbar formulieren.
2. Bei Tool-Empfehlungen offizielle Website- oder GitHub-Links angeben.
3. Wenn du mitarbeiten möchtest, nenne:
- Rolle
- Tech-Stack
- Verfügbarkeit pro Woche

## Regel zu öffentlich geteilten Ideen

In diesem Repository veröffentlichte Ideen gelten als öffentlich geteilt.  
Keine vertraulichen oder geschäftskritischen Informationen posten.

## Empfohlenes Maintainer-Triage

1. Wöchentlich:
- Labels setzen
- Duplikate zusammenführen
- Gelöste Issues mit Referenzen schließen

2. Für aktive Projekte:
- Teams auffordern, Meilensteine und Fortschritt im selben Issue zu aktualisieren

3. Bei fehlendem Kontext:
- `needs-info` setzen
- Nach längerer Inaktivität schließen, falls Infos fehlen
